Unit 12: Capstone Project

The capstone project brings together the main themes developed across the module, including object-oriented design, secure coding, software architecture, testing, dependency injection and design patterns.

For this artefact, I developed a banking system which builds on the earlier concurrency work from Unit 6 and extends it into a more complete object-oriented design. The project includes account management, transaction processing, authentication, monitoring and reporting functionality.

The implementation also demonstrates several of the design decisions explored during the module. Dependency Injection is used to support testing, monitoring behaviour can be changed without rewriting the core transaction logic, and authentication behaviour can be tested using a mock security logger. This helped connect the later units on TDD, Dependency Injection and Inversion of Control back to the practical code.

The README has been included to show how the project can be run and how the expected test results can be reproduced.
