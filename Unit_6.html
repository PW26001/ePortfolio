Unit 6: Concurrency and Parallelism

Unit 6 focused on concurrency and parallelism in object-oriented design. This moved the module from structuring code for maintainability into designing code that behaves correctly when multiple operations happen at the same time.

For the marked assessment, I developed a thread-safe banking system in Python. The system allowed multiple users to deposit, withdraw and check balances concurrently, while using synchronisation to protect shared account balances from race conditions.

The BankAccount class used a lock around deposit, withdraw and get_balance so that account values could not be modified by multiple threads at the same time. The wider system included a TransactionSimulator to simulate concurrent users, and a TransactionManager to coordinate transfers between accounts.

Deadlock prevention became an important part of the design once transfers between accounts were introduced. The TransactionManager acquired locks in account number order, preventing circular waits and allowing transfers to complete safely. The report also demonstrated that total funds were conserved during transfer testing, even when some transfers were denied because of insufficient funds.

Testing was a significant part of this unit. The code included unit tests for account creation, deposits, withdrawals, balance checking, overdraft protection, transaction simulation, transfer management and deadlock prevention. The report also included reproducible commands for running the tests and simulations with different values.

This assessment received 66%, a Merit. The feedback recognised the strength of the object-oriented design, thread-safety mechanisms, deterministic testing and validation approach. It also identified improvements I could make in future, including clearer references, more explanation of lock ordering, scalability analysis, diagrams and a README file.

This unit was useful because it showed that object-oriented design is not only about clean structure. In concurrent systems, design decisions directly affect correctness, reliability and safety.
