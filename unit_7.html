Unit 7: Secure Coding Practice

Unit 7 focused on secure coding practice.

At this point my focus when it came to OOP changed. Rather than seeing it only as a structure of best practice, or a pattern which enables extensibility at a later date, secure coding is about ensuring that whether through user error or malicious intent, all scenarios have a safe route which does not result in an error or unintended access.

The authentication task demonstrated this clearly. The original system stored passwords in plaintext, accepted weak passwords, did not validate input, did not limit failed attempts and did not log authentication activity.

This unit helped me understand security as something that should be factored into a system's design at an early stage, rather than retro-fitted afterwards, and the drawbacks of not doing so.
